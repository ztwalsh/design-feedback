| Term | Definition | Usage Guidelines | Related terms |  
| **SAML** | Security Assertion Markup Language (SAML) is used to authenticate and authorizate data between parties, most commonly used for single sign&hyphen;on (SSO). It allows users to authenticate once with an identity provider (IdP) and then access multiple service providers (SPs) without needing to re&hyphen;enter their credentials.  | Use “SAML” (always all caps)  when referring to it in the product. Use most commonly in conjunction with SSO (Example: SAML single sign&hyphen;on (SSO). Add a tooltip when possible to explain the concept. |  |  
| **scheduled** | Zap status that indicates a Zap run is scheduled to rerun due to an error, but autoreplay is enabled in your account. | Don’t capitalize unless used to start a sentence, heading, title, UI label, or standalone phrase.<br> |  |  
| **SCIM** | Automatically manage access to your account. | Always use with User provisioning, unless spacing does not allow for the full wording, User provisioning (SCIM). Use a tooltip to explain the concept when possible. |  |  
| **search action** | Action that finds existing data in your app to send to apps in other steps. | Don’t capitalize unless used to start a sentence, heading, title, UI label, or standalone phrase; when capitalized, use sentence case.<br> |  |  
| **search step** | Feature for finding existing data in connected apps. | Don’t capitalize unless used to start a sentence, heading, title, UI label, or standalone phrase; when capitalized, use sentence case.<br> |  |  
| **shared folders** | Folders that are shared with everyone in the account or specific users. | Don’t capitalize unless used to start a sentence, heading, title, UI label, or standalone phrase; when capitalized, use sentence case.<br> |  |  
| **shared Zap** | Zap that’s shared with team members, and they have permission to edit it. | Don’t capitalize “shared” unless used to start a sentence, heading, title, UI label, or standalone phrase; when capitalized, both words should be capitalized. |  |  
| **Single sign&hyphen;on (SSO)** | Allows access to multiple apps and services with one set of login credentials. | The first time it is introduced on a page, use the full word with the abbreviation in parentheses. Additional mentions can just use SSO. |  |  
| **SSL certificate** | A digital security certificate that encrypts data transferred between your browser and Zapier, ensuring your information stays private and secure during transmission. | “SSL” should always be in all caps. Use sentence case.  |  |  
| **step** | An individual trigger or action within a Zap. | Don’t capitalize unless used to start a sentence, heading, title, UI label, or standalone phrase.<br> |  |  
| **step notes** | A feature that allows users to write notes or documentation at the step level within a Zap. | Don’t capitalize unless used to start a sentence, heading, title, UI label, or standalone phrase; when capitalized, use sentence case.<br> |  |  
| **stopped** | Status that indicates that the run had an issue and could not be completed. <br>There are three sub&hyphen;statuses of stopped: errored, halted, and handled. | Don’t capitalize unless used to start a sentence, heading, title, UI label, or standalone phrase.<br> |  |  
| **Sub&hyphen;Zap** | Sets of actions that can receive data from other Zaps and send back the results of their actions. Instead of rebuilding the same steps across multiple workflows, you can create a Sub&hyphen;Zap once and use it within different Zaps. | Always capitalize and hyphenate. |  |  
| **success** | Status when a Zap runs without problems. | Don’t capitalize unless used to start a sentence, heading, title, UI label, or standalone phrase.<br> |  |  
| **super admin** | User role with additional permissions. | Only capitalize when referring to a specific super admin. For example, Super Admin Brian. Otherwise, do not capitalize unless used to start a sentence; in that case, use sentence case.   |  |
